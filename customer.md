# customer

## 핵심 사용자
- 현장 사용자
- 고객센터 민원 담당자
- 상품 관련 VOC 담당자

## 사용자 목표
- 할당받은 VOC를 빠르게 이해하고 답변 초안을 작성하고 싶다.
- 매번 매뉴얼과 과거 VOC 이력을 찾는 시간을 줄이고 싶다.
- 현장에서 바로 답변 가능한 문의는 즉시 처리하고 싶다.
- 바로 답변하기 어려운 건은 상품 관련 VOC 담당자에게 자연스럽게 넘기고 싶다.
- 금액 산출 방법이나 근거 문의에는 바로 참고 문서를 찾고 싶다.
- 실제 금액 산출 요청은 자동 처리하지 않고 상품팀 담당자 기준으로 안전하게 안내하고 싶다.
- 이전에 비슷했던 VOC 사례를 참고해 답변 품질을 높이고 싶다.

## 사용자 과제
1. VOC 내용을 읽고 어떤 이슈인지 파악한다.
2. 관련 상품정보를 함께 확인한다.
3. 현장 또는 고객센터에서 즉시 답변 가능한지 판단한다.
4. 금액 산출 방법이나 근거를 묻는 경우 관련 매뉴얼 위치를 찾는다.
5. 실제 금액 산출 요청이면 상품팀 담당자가 수작업 산출해야 하는 건인지 확인한다.
6. 현장 응대가 어렵다면 상품 관련 VOC 담당자에게 추가 확인을 요청한다.
7. 기존 VOC 중 유사한 케이스를 확인한다.
8. 이를 바탕으로 답변 초안을 작성해 고객 응대 또는 후속 전달에 활용한다.

## 주요 페인포인트
| 상황 | 불편 |
| --- | --- |
| 매뉴얼과 이력이 분산됨 | 어디를 먼저 봐야 할지 시간이 걸림 |
| VOC가 모호함 | 예전 사례가 없으면 답변 방향 잡기가 어려움 |
| 답변 품질 편차 | 담당자마다 문장과 근거 수준이 달라질 수 있음 |
| 검토 시간 부족 | 초안을 빨리 만들고 싶지만 근거 확인도 필요함 |
| 금액 산출 요청 포함 | 자동 제공이 가능한지 아닌지 판단 기준이 필요함 |
| 현장 응대 불가 건 발생 | 누구에게 넘겨야 하는지와 어떤 정보가 더 필요한지 판단이 필요함 |

## 기대 가치
- 답변 작성 시간 단축
- 근거 기반 답변 품질 향상
- 유사 VOC 재사용으로 일관성 확보
- 신규 담당자도 빠르게 참고 가능
- 현장 즉시응대 가능 건의 처리 속도 향상
- 상품 관련 VOC 담당자에게 추가 확인이 필요한 건을 안정적으로 연결 가능

## 사용 맥락
- 고객에게서 받은 상품 관련 VOC를 당일 또는 짧은 시간 안에 처리해야 하는 상황
- 상품명, 상품코드, VOC 내용이 함께 주어지는 상황
- 현장 사용자나 고객센터 민원 담당자가 먼저 응대하는 상황
- 현장 응대가 어려우면 상품 관련 VOC 담당자가 추가 확인하는 상황
- 최종 등록은 사람이 확인한 뒤 진행해야 하는 상황
- 금액 산출 방법/근거 문의와 실제 금액 산출 요청을 구분해야 하는 상황
