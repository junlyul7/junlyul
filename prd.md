# prd

## 1. Problem Statement
삼성생명에서 상품 관련 VOC를 응대하는 현장 사용자, 고객센터 민원 담당자, 상품 관련 VOC 담당자는 고객 문의에 대해 빠르게 답변 가능 여부를 판단해야 한다. 금액 산출 방법이나 근거를 확인할 때는 관련 문서와 매뉴얼 위치를 찾아야 하고, 현장에서 바로 답변 가능한 건은 즉시 응대해야 하며, 현장 응대가 어려운 건은 상품 관련 VOC 담당자가 추가 확인해야 한다. 실제 금액 산출 요청이 들어오면 상품팀 담당자가 수작업으로 산출해 제공해야 한다. 현재는 참고 문서 탐색과 사례 검색에 시간이 걸려 답변 작성 속도가 느려질 수 있다.

## 2. Objectives
- VOC 내용과 상품정보를 바탕으로 금액 산출 방법이나 근거를 확인할 수 있는 관련 문서 및 매뉴얼 위치를 안내한다.
- 기존 VOC DB에서 동일한 `상품군 + 이슈유형` 기준의 유사 사례를 검색한다.
- 현장이나 고객센터가 즉시 답변 가능한 건은 바로 응대할 수 있도록 지원한다.
- 현장 응대가 어려운 건은 상품 관련 VOC 담당자가 추가 확인할 수 있도록 지원한다.
- 유사 사례를 바탕으로 답변 초안을 생성한다.
- 매뉴얼은 답변 작성에 직접 사용하지 않고, 금액 산출 방법·근거 확인 및 위치 안내 용도로만 사용한다.
- 실제 금액을 산출해달라는 문의에는 자동산출 결과를 제공하지 않는다.
- 실제 금액은 상품팀 담당자가 수작업으로 산출해 제공해야 한다.
- 근거가 없으면 추측하지 않고 `근거 없음`으로 표시한다.

## 3. Non-goals
- 작성자 확인 없이 최종 등록 또는 전송을 수행하지 않는다.
- 근거 없는 답변을 자동 생성하지 않는다.
- 화면이나 로그에 키, 비밀값, 민감정보를 출력하지 않는다.
- 상담사 재안내용 최종본을 대신 확정하지 않는다.

## 4. Input Data
| 항목 | 설명 |
| --- | --- |
| 상품명 | VOC와 연결된 상품명 |
| 상품코드 | 상품 식별 코드 |
| VOC 내용 | 고객 문의 또는 이슈 내용 |
| 기존 VOC data | 유사 사례 검색용 데이터 |
| 접수 주체 | 현장 사용자, 고객센터 민원 담당자, 상품 관련 VOC 담당자 |

## 5. Output Data
| 항목 | 설명 |
| --- | --- |
| VOC 답변 초안 | 사람이 검토할 수 있는 초안 문장 |
| 금액 산출 관련 문서/매뉴얼 경로 | 금액 산출 방법 또는 근거 확인을 위해 참고할 문서 위치 |

## 6. User Flow
1. 사용자가 상품명, 상품코드, VOC 내용을 입력한다.
2. 시스템이 VOC를 `상품군 + 이슈유형`으로 분류한다.
3. 동일 분류의 유사 사례를 검색한다.
4. 현장이나 고객센터에서 즉시 답변 가능한지 판단한다.
5. 금액 산출 방법이나 근거 문의이면 관련 매뉴얼 위치를 안내한다.
6. 실제 금액 산출 요청이면 자동 제공 불가와 상품팀 담당자 수작업 산출 원칙을 안내한다.
7. 현장 응대가 어려우면 상품 관련 VOC 담당자가 추가 확인한다.
8. BizRouter LLM이 초안을 작성한다.
9. 사용자가 초안을 검토해 고객 응대 또는 후속 전달에 활용한다.

## 7. Functional Requirements
### 기능 1. 매뉴얼 위치 안내
- VOC 내용과 상품정보를 바탕으로 금액 산출 방법이나 근거를 확인할 수 있는 참고 문서 및 매뉴얼 위치를 안내해야 한다.
- 매뉴얼이 없거나 불명확하면 `근거 없음`으로 표시해야 한다.
- 매뉴얼명만이 아니라 실제 경로 또는 위치를 함께 제시해야 한다.
- 매뉴얼 내용은 답변 문장 작성 근거로 사용하지 않고, 금액 산출 방법·근거 확인용으로만 사용한다.

### 기능 2. 금액 산출 요청 처리
- 실제 금액을 산출해달라는 문의에는 자동산출 결과를 제공하지 않아야 한다.
- 시스템은 `상품팀 담당자가 수작업으로 산출해 제공해야 한다`는 원칙을 명시해야 한다.
- 금액 산출이 필요한 경우에도 임의의 숫자, 예시 금액, 추정 금액을 생성하지 않아야 한다.

### 기능 3. 현장 즉시응대 / 추가 확인 분기
- 시스템은 현장 또는 고객센터에서 즉시 답변 가능한 건과 추가 확인이 필요한 건을 구분할 수 있도록 도와야 한다.
- 즉시 답변이 어려운 건은 `상품 관련 VOC 담당자 추가 확인 필요` 상태로 넘길 수 있어야 한다.

### 기능 4. 유사 VOC 검색 및 초안 작성
- VOC 내용을 바탕으로 기존 VOC DB에서 `상품군 + 이슈유형`이 같은 사례를 검색해야 한다.
- 검색 결과를 바탕으로 답변 초안을 작성해야 하며, 답변 문장은 유사 VOC를 참고해 구성해야 한다.
- 검색 결과가 0건이면 단정하지 말고 상담사 연결 또는 추가 확인 필요로 표시해야 한다.
- LLM 실패 시 `잠시 후 다시`로 안내해야 한다.
- 금액 관련 실제 수치가 필요한 문의에는 자동산출 불가 원칙을 함께 표시해야 한다.

## 8. Processing Rules
- 입력: VOC 내용
- 처리: 분류 → 검색 → LLM
- 출력: 답변 + 근거
- 근거가 없으면 추측하지 않는다.
- 현장 즉시응대 가능 여부를 먼저 판단한다.
- 금액 산출 방법이나 근거 문의에는 관련 문서와 매뉴얼 위치를 안내한다.
- 현장 응대가 어려운 경우에는 상품 관련 VOC 담당자 추가 확인으로 넘긴다.
- 실제 금액 산출 요청에는 자동산출 결과를 제공하지 않고 상품팀 담당자 수작업 산출 원칙을 안내한다.
- 매뉴얼은 금액 산출 방법·근거 확인에만 사용하고, 답변 문장 작성에는 반영하지 않는다.

## 9. Definition of Done
- 매뉴얼 위치 안내 또는 자동산출 불가 안내와 VOC 답변 초안이 작성되면 완료로 본다.
- 참고할 매뉴얼과 VOC 답변을 찾지 못하면 `근거 없음`으로 표시한다.
- 초안은 상담사 전달 전 단계까지만 지원한다.

## 10. Failure Handling
| 상황 | 처리 |
| --- | --- |
| 검색 0건 | 근거 없음, 단정 금지, 상담사 연결 안내 |
| LLM 실패 | 잠시 후 다시 안내 |
| 근거 부족 | 추측 금지, 추가 확인 필요 표시 |
| 현장 즉시응대 어려움 | 상품 관련 VOC 담당자 추가 확인 필요 |
| 실제 금액 산출 요청 | 자동산출 불가, 상품팀 담당자 수작업 산출 안내 |

## 11. Approval and Review
- 최종 등록은 작성자의 확인 이후에만 가능하다.
- 시스템은 초안 작성까지만 수행한다.
- 상담사 전달용 문안은 사람이 검토한 뒤 확정한다.

## 12. Success Criteria
- 사용자가 VOC 초안 작성 시간을 줄였다고 느낀다.
- 매뉴얼 위치와 유사 사례를 빠르게 찾을 수 있다.
- 현장이나 고객센터가 즉시 답변 가능한 건을 더 빨리 처리할 수 있다.
- 현장 응대가 어려운 건이 상품 관련 VOC 담당자에게 자연스럽게 연결된다.
- 답변 초안이 근거 중심으로 정리된다.
- 실제 금액 산출 요청을 자동 처리하지 않고 올바르게 상품팀 담당자 검토 단계로 넘길 수 있다.
